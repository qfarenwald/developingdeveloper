My friend required me to use the terminal to move around the app project and to push and pull files from GitHub. Below is a list of the initial commands I've been using. Oh, but before we move on, it is important to know the structure of files and how the terminal navigates them. Files and folders are set up like a tree. For example, the desktop is like the trunk and each folder on the desktop top is its own branch, and any folders or files in those folders create even more branches off those initial branches, and so on. Knowing where exactly you are located in the terminal and how to move around is very important.

Basic Terminal Commands

  • mkdir - makes directory
  • ls - lists the files in the current working directory
  • cd - changes directory to the start of the 'tree'
  • cd .. - changes directory just to the previous directory
  • cd [folder/file name] - changes directory forward into the next specified directory
  • atom . - opens files where you are located in the directory in the atom program
  • [command] t - opens another terminal, this has been necessary to open and run my server

Basic Git Commands

  • git status - shows if and what files have been changed
  • git checkout - change one branch to the other
  • git checkout -b [name] - create (build) a new branch, should always be done after a push
  • git branch -d [name] - delete a branch, keep files clean and delete branches you don't need
  • git branch - shows all branches, the branch you are on is green
  • git stash - saves unfinished changes that you can reapply anytime so you don't have to commit half-done work
  • git init - creates a new git repository

Get Files to Git

  • git add . - adds all files where you are currently located in directory to be ready to commit
  • git add [name of file] - adds that exact file to be ready to commit
  • git commit -m "[message]" - commits file to be pushed up into the cloud, message is mandatory and should be addressed to the person(s) you are pushing the file too with some detail about what you are pushing, often times curse words exist in these messages and this handy little program captures all of them, haha
  • git push - pushes the file to the cloud, if it fails, just copy the text it prompts you to use and hit enter
  • git pull - pulls the file down from the cloud, always complete after pushing to the cloud to ensure local master files match the remote cloud master files, make sure you are in the master branch

Server Commands

  • rails s - starts server
  • control c - closes server

Similar to my lists above, git - the simple guide provides a clear flow of how and when to use git commands.

Many of my handy terminal commands deal with git branches. A successful Git branching model is a helpful article on how best to stucture and work within your branches.
